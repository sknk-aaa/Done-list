## 1. アプリ概要

「やること / やったこと」を記録する **ログ型タスクアプリ**。ToDo としても使えるが、主役は「後から振り返れること」。

- ストア展開：**英語をプライマリ、日本語をローカライズ**（単一ビルド、ロケールで言語切替）
- 主役は振り返り（月ビュー）。ToDo 的な「繰り越し」などの機能は持たない。
- **アプリ名：未定（保留）**。決定までは仮名で進める。
- **コアはあくまで「シンプルなやること/やったことリスト」**。時刻・通知・各種設定（時刻表示、左右スワイプの挙動など）は**任意の補助機能**で、使わなくてもコア体験（サッと記録して振り返る）が完結する。これらをデフォルト前提で設計しない（時刻なし・通知なしでも自然に使えること）。

### テーマ / 配色
- **ライトテーマに統一**（白背景・区切り線ベースのリスト UI。カードの影は使わない／極力薄く）。タブ1・タブ2 どちらもライト。
- アクセントカラー：**ティール `#14B8A6` で確定**。ユーザーが設定で変更可能（後述 settings）。FAB・選択中の日付・今日ハイライト・「保存」「タグを編集」等の主要操作に使う。
- **破壊的操作（削除）だけは赤系**を別枠で使用。
- 注意：**タスクの完了/未完了チェックの色はアクセント色ではなく「そのタスクのタグ色」**を使う（§4 リスト部参照）。アクセント色とタグ色は役割が別。

### マネタイズ（初版）
- **無料版**：全機能（記録・タブ1・タブ2・タグ・絞り込み・ダークモード・**時刻表示・通知**等）を利用可能。画面下にバナー広告を表示。
- **Pro版（初版）で解放されるのは次の1つのみ**：
  1. **広告非表示**
- ダークモード・時刻表示・通知は**無料**で提供する（Pro ではない）。
- **iCloud 連携（バックアップ／複数端末同期）は将来提供予定**。初版では実装しない／Pro の課金項目にも含めない（実体のない機能を課金に紐づけると審査リスクがあるため）。実装する際はローカル SQLite の同期方式を別途設計する（後付けは重いので注意）。
- 上記以外の機能差は初版では設けない。

## 3. データモデル（確定・Drizzle スキーマ）

> 設計判断には理由を併記。**カラム構成・型・制約は変更しないこと**。

```typescript
import { sqliteTable, integer, text } from 'drizzle-orm/sqlite-core';
import { relations } from 'drizzle-orm';

// ── tags ──────────────────────────────
export const tags = sqliteTable('tags', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  name: text('name').notNull(),
  color: text('color').notNull(),                          // プリセット10色のhex
  sortOrder: integer('sort_order').notNull().default(0),   // タグ一覧/絞り込みシートの並び
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
});

// ── items（タスク＝やること/やったこと）──
export const items = sqliteTable('items', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  title: text('title').notNull(),
  memo: text('memo'),                                      // 任意
  date: text('date').notNull(),                            // 'YYYY-MM-DD' ローカル日付（理由は下記）
  time: text('time'),                                      // 'HH:MM' ローカル時刻・任意（null=時刻なし）
  notifyEnabled: integer('notify_enabled', { mode: 'boolean' }).notNull().default(false), // 通知を飛ばすか
  isCompleted: integer('is_completed', { mode: 'boolean' }).notNull().default(false),
  sortOrder: integer('sort_order').notNull().default(0),   // 日付スコープ内の並び
  tagId: integer('tag_id').references(() => tags.id, { onDelete: 'set null' }), // タグ削除→null
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
  completedAt: integer('completed_at', { mode: 'timestamp' }),  // 完了時刻・未完了ならnull
});

// ── settings（アプリ設定・単一行運用）──
export const settings = sqliteTable('settings', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  accentColor: text('accent_color').notNull().default('#14B8A6'),  // ティール初期値
  locale: text('locale'),                                  // null=端末ロケール自動、手動上書き時のみ値
  launchCount: integer('launch_count').notNull().default(0),       // アプリ起動回数（レビュー判定用）
  reviewRequested: integer('review_requested', { mode: 'boolean' }).notNull().default(false), // 自動レビュー要求済みか
  showTime: integer('show_time', { mode: 'boolean' }).notNull().default(true),  // リストに時刻を表示するか（ドロワーで切替・既定ON）
  swipeAction: text('swipe_action').notNull().default('date'),     // 左右スワイプの挙動 'date'(日付移動) | 'tab'(タブ切替)。既定 date
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
});

export const itemsRelations = relations(items, ({ one }) => ({
  tag: one(tags, { fields: [items.tagId], references: [tags.id] }),
}));
export const tagsRelations = relations(tags, ({ many }) => ({
  items: many(items),
}));
```

### 設計理由（重要）
1. **`date` は `text('YYYY-MM-DD')`、`time` は別カラム `text('HH:MM')`（どちらもローカル基準・分離して持つ）**：日付と時刻を1つの timestamp にまとめるとタイムゾーン事故（深夜のタスクが前日にズレる等）の原因になる。日付は文字列で持てばカレンダー集計（`GROUP BY date`）が素直に効き、ズレない。時刻は任意（null=時刻なし）で、表示と通知にのみ使う。通知をスケジュールする時だけ `date`＋`time` を端末ローカルの日時に組み立てる（`expo-notifications` が端末のタイムゾーンで正しく発火する。各国の正しい現地時刻で鳴る）。
2. **`notifyEnabled`**：通知を飛ばすかをタスク単位で持つ。通知 ID はタスクの `id` から決定的に生成（別カラム不要）。スケジュール対象は「未来の日時 かつ 未完了 かつ `notifyEnabled=true` かつ `time` が非null」のみ。完了にした／過去になった／削除した／`notifyEnabled` を切った場合は対応する通知をキャンセルする（§通知ロジック参照）。
2. **`onDelete: 'set null'`**：タグ削除時、タスクは残し `tagId` を null（＝タグなし・色なし）に戻す。**連鎖削除は絶対にしない**。記録の保全がこのアプリの最重要事項。
3. **`completedAt`**：画面には出さないが将来の振り返り機能のため確保。`isCompleted`（状態）と `completedAt`（いつ完了したか）は別情報なので両方持つ。完了時に時刻記録、未完了に戻したら null。
4. **`sortOrder`**：items は日付スコープ内の並び（両タブで共有）、tags は絞り込みシート用。
5. **settings は単一行運用**。`locale` が null のときは端末ロケールに自動追従、ユーザーが言語を手動固定したときだけ値を入れる。`launchCount`・`reviewRequested` はアプリ内レビュー判定用（§11 参照）。

### 完了/未完了トグル時の注意（必読）
- 完了/未完了を切り替えるとき触るのは **`isCompleted` と `completedAt` のみ**。
- **`sortOrder` は絶対に触らない**（完了しても並び順は動かさない＝後述「案X」）。

## 4. 画面構成：下部2タブ

### タブ1：デイリー（操作画面）

選択した1日のタスクリストが主役。タスクの追加・完了・編集はここで行う。**タブ1にはカレンダー（週/月）を表示しない**（月の俯瞰はタブ2が担うため重複を避ける）。

**ヘッダー（上段）**
- **左：ハンバーガーメニュー（≡）**
- **右：絞り込みアイコン（じょうご型）**
- ※「2026 / 6月」のような独立した年月ヘッダーは置かない（下記 日付ナビに集約）。
- ※追加ボタンは上段に置かない（FAB のみ。後述）。

**日付ナビ（ヘッダー下・1〜2段）**
- 中央に現在の日付。表記は**相対表記を基本**：当日「今日」、前後「昨日」「明日」、それより離れた日は曜日付き絶対表記「6月19日(金)」。実装上は実日付（例：`6月19日(金)`）を主役に出し、当日であることは下段の「今日」ラベルで示してもよい（モック準拠）。
- 1段目に日付＋ドロップダウン（▾）、2段目に小さく **「今日 · 4/7完了」** の達成サマリーを表示。
  - 達成サマリーは **「完了数 / その日の総数」**（分母つき）。
  - フィルタ中は**絞り込み後の件数**で数える（カレンダーバッジや月ビューと整合）。
- 左右の **「‹」「›」** で前日/翌日へ移動。
- **日付部分タップで「今日」に戻る**（今日にいるときは何もしない）。独立した「今日」ボタンやカレンダーピルは作らない。
- リスト全体を**左右スワイプでも前日/翌日に移動**できる（矢印はその視覚的ヒント）。

**リスト部（選択日のタスク）**
- 各行：チェック丸 ＋ タスク名 ＋ 下段（タグドット＋タグ名、メモアイコン）。**カードではなく区切り線**で各行を分ける（影は使わない）。
- **チェック丸タップ＝完了/未完了トグル**。
  - **チェックの色は「そのタスクのタグ色」**（アクセント固定ではない）。完了＝タグ色で塗ったチェック、未完了＝タグ色の輪郭（中抜き）。
  - タグなしのタスクはグレーで表示。
  - 当たり判定は丸の周囲に十分なパディングを取り、行タップ（編集）と明確に分離する。
- **行（テキスト部分）タップ＝編集画面へ遷移**。
- **タグ表示**：タスク名の下段に、🏷️タグアイコン＋タグ名を**そのタグの色**で表示（行全体は塗らない）。タグなしの場合は何も出さない。
- **メモがある場合**：下段に小さなメモアイコンを表示。
- **時刻表示**：`settings.showTime == true`（既定ON）のとき、各行の右側にそのタスクの `time`（"08:00" など）を表示。`time` が null のタスクは何も出さない。`showTime == false` のときは行に時刻を出さない。
- **通知アイコン**：`notifyEnabled == true`（かつ通知対象＝未来・未完了）のタスクは、時刻の横にベルアイコン（アクセント色）を表示。
- **長押しドラッグでリスト並び替え**（`sortOrder` を更新）。並び順は日付スコープ内で保持。
- **新規タスクは末尾に追加**。完了しても並び順は動かさない（§3 案X）。
- 右下：**FAB（＋・ティール）で新規タスク作成**。リストを下までスクロールしても FAB は固定表示。

**空状態**：その日にタスクが無いときの空表示を用意（文言は i18n）。

### タブ2：月ビュー（振り返り）

1ヶ月を見渡す 7×6 グリッド。各日セルにタスクを**タグ色の帯**で表示。

- **1セル最大4件**まで帯表示。5件目以降は **「+N」省略インジケータ**を4件目の下に表示。
- 帯の並びは **`sortOrder` 順**（タブ1と共有。完了/未完了を混ぜたまま並べる）。
- **完了＝タグ色で塗り、未完了＝タグ色の枠線のみ**。
  - **タグなし（色なし）**：グレーをデフォルト色として使用（塗り/枠線ルールはグレーで成立させる）。
  - 視認性注意：未完了の枠線がライト背景で潰れないよう、枠線は十分な太さ・コントラストを確保。
- **日（セル）タップ → 下からその日の詳細リストがせり上がる**（ボトムシート的）。▼ で閉じる。詳細リストの各行から編集に飛べる。
- **左右スワイプで前月/翌月へめくる**（`settings.swipeAction` に関わらず、月ビューでは常に月めくり）。年月は「2026 / 6月 ▾」からも変更可。タブ1へ戻るのは下部タブバー。
- **ヘッダー**：左に「2026 / 6月 ▾」、右に絞り込みアイコン・ハンバーガー（≡）。絞り込みアイコン・ハンバーガーはタブ1と同じ位置・同じ挙動（フィルタ状態は共有）。年月表示はタブ2では残す（月の俯瞰ビューなので）。

## 5. タグ（カテゴリ）管理

- **1タスクにつきタグは1つ**（単一選択）。
- **専用画面**で管理（タグ名入力＋色パレットを同一画面に。簡易ダイアログにはしない）。
- **プリセット10色**から選択。
- **同じ色を複数タグで選択可**（ユーザー任せ・制約なし）。
- タグの新規作成・名前変更・色変更・削除が可能。
- **タグ削除時：そのタグが付いたタスクは消さず、`tagId` を null（タグなし・色なし）に戻す**。
- タグ一覧・絞り込みシートでの並びは `tags.sortOrder`。

## 6. 絞り込み（フィルタ）

- **入口**：ヘッダー右の**絞り込みアイコン（じょうご型）**をタップ → **タグ選択シート**（ボトムシート）。タブ1・タブ2 とも同じ。
- シート内容：
  - 先頭に **「すべて表示」**（＝フィルタ解除）。
  - **「タグなし」** 項目（`tagId IS NULL` のタスクを拾う）。
  - 各タグ（色付き）。単一選択。
- **解除**：シート内「すべて表示」を選ぶ（解除専用 UI は他に作らない）。
- **フィルタ中の表示**：ヘッダーの絞り込みアイコンを**必ず状態変化させる**（色付き or バッジ）。これが唯一のフィルタ中インジケータなので必須。
- **適用範囲**：
  - タブ1・タブ2 で**フィルタ状態を共有**。
  - 反映先：タブ1 の達成サマリー（「○/○完了」）、月ビュー（タブ2）の**帯**および「+N」の N。
  - **「+N」の N は絞り込み後の件数で数える**（フィルタ後データと表示ロジックは同じソースを見ること）。
  - タブ1のリストは**選択日のぶんだけ**表示（案B：絞り込んでもリストは選択日のまま、横断一覧は出さない）。

## 7. 新規タスク作成 / 編集

### 新規タスク作成（ボトムシート）
FAB（＋）から下からせり上がるボトムシートで作成する。
- ヘッダー：左「キャンセル」／中央「新規タスク」／右「保存」（保存はアクセント色）。
- **やったこと**（タスク名）入力欄。プレースホルダ「やったことを入力」。
- **タグ**：色ドット付きのチップを横に並べて単一選択。右上に「タグを編集」（→ タグ管理画面）。
- **時刻**：時刻ピッカー（任意・未設定可）。
- **通知を飛ばす**：トグル（タスク単位）。ON にすると `notifyEnabled=true`。`time` が未設定のときは通知を出せない旨を示す／時刻設定を促す。初回 ON 時に通知権限をリクエスト（§通知ロジック）。
- **日付**：日付ピッカー（デフォルトはそのとき選択中の日）。
- **メモ**：任意・複数行。
- ※時刻は任意。設定しなければ「時刻なしのログ」として記録される。

### 編集画面（リスト行タップ・ボトムシート）
- ヘッダー：左「キャンセル」／中央「タスクを編集」／右「保存」。
- **タスク名**（必須）
- **タグ**：チップ単一選択＋「タグを編集」
- **時刻**：時刻ピッカー（任意）
- **通知を飛ばす**：トグル（`notifyEnabled`）
- **日付**：日付ピッカー
- **メモ**：任意・複数行
- 下部に**「このタスクを削除」**（赤・物理削除）。
- 完了/未完了の変更は §3 の注意（`sortOrder` を触らない）に従う。削除・完了・時刻/通知変更時は対応する通知を更新/キャンセルする（§通知ロジック）。

## 8. ジェスチャー競合の解決（実装上の必須事項）

タブ1は同一画面に複数ジェスチャーが同居する：横スワイプ（前日/翌日の移動）／長押しドラッグ（並び替え）／タップ（完了・編集）。

- `react-native-gesture-handler` の `simultaneousHandlers` / `waitFor` で競合解決を**最初から設計する**こと。
- 横スワイプ（日付移動）と長押しドラッグ（並び替え）が取り合わないようにする。ドラッグ中は横スワイプを無効化するなど線引きを明確に。
- チェック丸タップと行タップ（編集）の当たり判定を明確に分離する。
- ※タブ1にカレンダーの折りたたみ（縦スワイプ開閉）は無い。月の俯瞰はタブ2。
- 左右スワイプの割り当て（**タブ1のみ** `settings.swipeAction` に従う）：`'date'`＝日付移動／`'tab'`＝タブ1→タブ2 切替。`'tab'` のとき日付移動は矢印に委ねる。**タブ2の左右スワイプは設定に関わらず常に月めくり**なので、タブ切替スワイプとは競合しない（§11 注記参照）。

## 9. パフォーマンス

- 集計（タブ1 の日次「○/○完了」、タブ2 月グリッドの各日のタスク）は、毎回全件 JS ループで数えない。
- Drizzle で **`count()` + `groupBy(date)`** の集計クエリ、月グリッド用の「日付ごとのタスク取得」クエリを分けて用意する。
- フィルタ時はそれらのクエリにタグ条件（`tagId = ?` または `tagId IS NULL`）を足す。

## 9.5 通知ロジック（`expo-notifications`）

- **スケジュール対象**：`time` が非null かつ `notifyEnabled=true` かつ 未完了 かつ 日時が未来、のタスクのみ。
- **発火時刻**：`date`＋`time` を端末ローカルの日時として組み立ててスケジュール（端末のタイムゾーンで正しく発火する。各国の現地時刻で鳴る）。
- **通知 ID**：タスクの `id` から決定的に生成（タスク1件＝通知1件で対応づけ。別カラム不要）。
- **更新・解除のタイミング**：以下のとき、そのタスクの通知を再スケジュール or キャンセルする。
  - 完了にした → キャンセル
  - 未完了に戻した（かつ未来・notifyEnabled・time有）→ 再スケジュール
  - `time` 変更・`date` 変更・`notifyEnabled` 切替 → 再スケジュール or キャンセル
  - タスク削除 → キャンセル
  - 過去日時になった → 発火しない（スケジュールしない）
- **権限**：初回に「通知を飛ばす」を ON にしたとき通知権限をリクエスト。拒否された場合はトグルを ON にできない旨を案内（設定アプリへの導線があると親切）。
- 通知は**無料機能**（Pro 限定ではない）。

## 10. アプリ内レビュー

レビュー導線は2系統。混同しないこと。

### 手動レビュー（ドロワーの「レビューして応援する」）
- タップしたら **App Store のレビュー記入ページに直接遷移**（App Store URL に `?action=write-review` を付けて開く）。文章まで書ける画面に飛ばす。
- これは `requestReview`（ネイティブダイアログ）ではない。ユーザーが能動的に応援する導線なのでフルレビュー画面へ。

### 自動レビュー（アプリ内ネイティブ星ダイアログ）
- `expo-store-review` / `SKStoreReviewController` の **`requestReview`**（その場で完結する星ダイアログ。ストアには飛ばさない）。
- **発動条件（すべて満たし、かつトリガー発生時に1回だけ呼ぶ）**：
  1. 記録した日数 ≥ 5日（`items` の distinct な `date` の数。継続利用の指標）
  2. `settings.launchCount` ≥ 5
  3. トリガー＝**タスクを完了にした瞬間**（全完了かどうかは問わない）
  4. `settings.reviewRequested == false`
- 発動したら `settings.reviewRequested = true` にし、**自動では二度と出さない**。
- 注意：`requestReview` は呼んでも OS が表示可否を最終判断する（年間回数制限あり）。開発側は「良い瞬間に呼ぶ」ことに集中すればよく、出し過ぎ事故は OS が防ぐ。
- `launchCount` はアプリ起動時にインクリメントする。

## 11. ドロワー（メニュー / 設定画面）

ヘッダー左のハンバーガー（≡）から開く。構成：
- 先頭：アプリ名＋プラン表示（Free / Pro）
- **Pro 版にアップグレード**（→ 課金画面）。初版 Pro の内容は §「マネタイズ」に従う＝**広告非表示のみ**。
- 設定：
  - **時刻を表示**：トグル（`settings.showTime`。既定 ON）。OFF にするとリストの時刻表示を隠す（§4 リスト部）。
  - **左右スワイプ時の挙動**：「日付の移動」/「タブ切替」のトグル（デフォルトは日付の移動）。
- サポート：使い方 / よくある質問 / 不具合・要望を報告 / **レビューして応援する**（§10 手動レビュー）/ オンボーディングを見る
- 末尾：アプリ名＋バージョン

> ⚠️ **「左右スワイプ時の挙動」設定（`settings.swipeAction`、既定 'date'）**：この設定が効くのは**タブ1のみ**。
> - タブ1で `'date'`：左右スワイプ → 前日/翌日へ移動（§8 と一致）。
> - タブ1で `'tab'`：左右スワイプ → タブ1→タブ2 へ切替。この場合タブ1の**日付移動は日付ナビの「‹ ›」矢印のみ**になる（スワイプは日付移動に使わない）。
> - **タブ2（月ビュー）は `swipeAction` に関わらず、左右スワイプは常に「前月/翌月」めくり**。タブ2からタブ1へ戻るのは下部タブバーで行う（スワイプ戻りは設けない）。これによりタブ切替スワイプと月めくりの競合が原理的に起きない。
> ⚠️ デザインモックの Pro 説明に「タグ無制限・テーマ変更」が含まれているが、**初版 Pro は広告非表示のみ**。タグ無制限・テーマ変更は提供しないので、モックのその表記は無視すること。

## 12. 開発順序（推奨）

1. データモデル / マイグレーション（§3）
2. 共通レイアウト（ヘッダー・タブバー・テーマ / アクセントカラー）

4. 編集画面 / タグ管理画面
5. 絞り込み
6. タブ2（月ビュー）
7. ドロワー / アプリ内レビュー
8. i18n（en プライマリ / ja ローカライズ）
