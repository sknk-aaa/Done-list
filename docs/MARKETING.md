# Marketing

toDoneのストア訴求、ASO、スクリーンショット、SNS/動画台本、価格/広告/Pro導線AB案を蓄積する場所。現在の作業状態は `docs/HANDOFF.md` に置く。

## 目的

- App Storeで「普通のToDo」との違いを短時間で伝え、毎日使う理由とレビュー/収益化につなげる。
- 実装相談でも、初見の伝わり方、ストアCV、レビュー、広告/Pro導線、継続利用への影響を同時に見る。

## 基本方針

- コア訴求は「やること」だけでなく「やったこと」が残ること。
- 月ビューで積み上げが見える、今日の記録をサッと残せる、タグで生活の流れを振り返れる、という価値を前面に出す。
- ToDoアプリの高機能競争に寄せすぎない。静かで、毎日開いても疲れず、記録を続けやすい体験を売る。
- 日本語では「やったことが残る」「今日できたことを見返せる」、英語では `toDone` と done-log / task journal 的な訴求を検討する。

## 避けること

- GTDやプロジェクト管理のような重いタスク管理アプリに見せること。
- 通知・時刻・細かい設定を主役にしすぎること。
- 広告/Pro導線が記録体験を邪魔しているように見える配置。
