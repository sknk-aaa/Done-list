# デザイン解釈と SPEC との相違（確定）

仕様の正は [../SPEC.md](../SPEC.md)。ビジュアル/挙動の正は `../to done/`（`To Done.html` が visual source of truth、`handoff/` に token・画面仕様・スクショ）。
両者が食い違う場合は **デザインを優先**する（ユーザー指示）。ただしデータモデル・通知・レビュー・課金内容など「デザインに現れない仕様」は SPEC に従う。

以下は実装にあたり確定した相違点。

| 項目 | SPEC | 採用（デザイン優先） |
|---|---|---|
| アクセント色 | `#14B8A6` | **`#48C1A8`**（`handoff/theme.ts`） |
| 絞り込みのタグ | 単一選択＋「すべて表示/タグなし」 | **ステータス(すべて/完了/未完了)＋タグ複数選択（既定で全選択、外すと絞り込み）** |
| 月チップの色 | タグ色 | タグ色（採用）。モックの名前別色（買い出し=灰/通院=赤）は、デイリー（買い物=橙）と矛盾する作り込みデータのため**非採用** |
| デイリーの日付移動 | 「‹ ›」矢印 | 矢印は置かず、**横スワイプ＋日付タップ(ポップオーバー)＋「↩今日へ」**（モック準拠） |
| 月セルのタップ | 詳細ボトムシートがせり上がる | **その日付でデイリーへ遷移**（HTML 準拠。詳細シートは未使用） |
| Add/Edit のフィールド順 | 名前→タグ→時刻→通知→日付→メモ | 同左（`handoff/03-screen-specs.md` §4 準拠）。時刻/通知は「時刻を表示」OFF で非表示 |
| Pro の内容 | 広告非表示のみ | 広告非表示のみ（モックの「タグ無制限・テーマ変更」表記は無視＝SPEC §11 の明示指示） |

## データモデル / ロジック（SPEC 準拠）

- スキーマは SPEC §3 のとおり（[../src/db/schema.ts](../src/db/schema.ts)）。`date`('YYYY-MM-DD')と`time`('HH:MM')は分離。
- 完了トグルは `isCompleted` と `completedAt` のみ更新、`sortOrder` は触らない。
- 通知は `time` 非null・`notifyEnabled`・未完了・未来 のみ対象（§9.5）。
- アプリ内レビューは distinct 記録日数≥5・launchCount≥5・完了トグル時・未要求 で1回（§10）。
